# AI Translator

Умный переводчик для macOS с поддержкой любых OpenAI-совместимых API
(OpenWebUI, OpenAI, Ollama, OpenRouter, Anthropic-compatible шлюзы и т.п.).

## Возможности

- Перевод между 16 языками + автоопределение исходного.
- Подключение к любым OpenAI-совместимым моделям через профили.
- Кастомные промпты для разных стилей перевода (буквальный, литературный,
  технический, упрощённый и т.д.).
- Иконка в menubar, быстрый popover и полноразмерное окно переводчика.
- Два режима «быстрого перевода» выделенного текста:
  - **Быстрый перевод в окне** — открывает окно с готовым результатом.
  - **Перевод на месте** — заменяет выделенный текст переводом прямо в
    активном окне (поддерживается в любом приложении, где работает
    `⌘C`/`⌘V`).
- Автосохранение и восстановление содержимого буфера обмена.

## Горячие клавиши

| Действие | По умолчанию |
| --- | --- |
| Открыть окно переводчика | `⌘O` |
| Открыть настройки | `⌘,` |
| Выйти | `⌘Q` |
| Быстрый перевод выделенного текста (в окне) | настраивается, по умолчанию `⌘⇧T` |
| Перевод выделенного текста на месте (с заменой) | настраивается, по умолчанию `⌘⇧T` |
| Отмена текущего перевода на месте | повторное нажатие той же горячей клавиши |

Поскольку оба «быстрых» действия используют одну и ту же комбинацию по
умолчанию, в настройках их можно переназначить независимо. Если комбинации
совпадают — приоритет получает «перевод на месте».

## Перевод на месте

1. Выделите текст в любом приложении (Telegram, браузер, IDE, Mail, …).
2. Нажмите настроенную горячую клавишу.
3. Приложение симулирует `⌘C`, отправляет текст в активную модель, кладёт
   результат в буфер, симулирует `⌘V` и восстанавливает исходный буфер.

Можно включить **«использовать собственные настройки»** — тогда перевод на
месте всегда выполняется с заранее заданными языком/стилем. Если выключено,
используются последние настройки, выбранные в окне переводчика.

### Автоматический обмен направлением

Дополнительно можно включить **«Автоматический обмен по определению языка»**.
В этом режиме приложение:

1. Определяет язык выделенного текста локально через Apple Natural Language
   framework (без обращения к сети).
2. Ищет в списке активных языковых пар ту, где есть найденный язык.
3. Переводит текст в **другой** язык этой пары.

По умолчанию активна одна пара `ru ↔ en`: русский текст переводится на
английский, а английский — на русский. Можно добавить любые другие пары
(ru ↔ de, en ↔ uk и т.д.) или отключать их по отдельности. Если язык
выделенного текста не попадает ни в одну активную пару, приложение
использует обычные настройки `source/target`.

Для работы требуется разрешение **Универсального доступа** (Accessibility) —
приложение запросит его при первом запуске.

## Структура проекта

- `AppAI_TranslatorApp.swift` — точка входа SwiftUI-приложения.
- `AppAppDelegate.swift` — статус-бар, окна, общий `TranslationService`.
- `AppAppDelegate+Hotkeys.swift` — глобальные хоткеи, быстрый перевод и
  перевод на месте.
- `ManagersSettingsManager.swift` — хранение и миграция настроек, профилей и
  параметров in-place перевода.
- `ServicesTranslationService.swift` — HTTP-клиент для OpenAI-совместимых API.
- `AI Translator/ModelsLanguagePair.swift` — модель двунаправленной языковой
  пары для in-place перевода.
- `AI Translator/UtilitiesLanguageDetector.swift` — обёртка над
  `NLLanguageRecognizer` для определения языка.
- `AI Translator/Views*` — SwiftUI-вьюхи (окно, popover, настройки).
- `AI Translator/ViewModelsTranslationViewModel.swift` — общий ViewModel
  логики перевода.

## Сборка

Открыть `AI Translator.xcodeproj` в Xcode 15+ и запустить. Минимальная цель —
macOS 14 (используется новый Observation framework).
